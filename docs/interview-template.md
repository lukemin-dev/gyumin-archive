# Interview Notes Template

이 템플릿은 다른 GitHub 저장소에도 복사해서 사용할 수 있습니다.

## 1. 프로젝트 한 줄 설명

이 프로젝트는 무엇을 해결하기 위해 만들었는가?

## 2. 처음 문제

- 어떤 불편함, 오류, 반복 작업이 있었는가?
- 왜 직접 만들거나 수정해야 했는가?
- 사용자가 겪는 문제는 무엇이었는가?

## 3. 내가 한 지시 또는 목표

- 어떤 기능을 추가하려고 했는가?
- 어떤 오류를 해결하려고 했는가?
- 어떤 문서나 배포 구조를 정리하려고 했는가?

## 4. 구현 과정

- 어떤 파일을 수정했는가?
- 어떤 구조를 선택했는가?
- 왜 그 방식을 선택했는가?
- 기존 구조와 충돌하지 않게 어떻게 연결했는가?

## 5. 막혔던 점

- 어떤 에러가 발생했는가?
- 배포, 의존성, 권한, API 제한, 데이터 구조 문제 중 무엇이었는가?
- 처음에 추측한 원인과 실제 원인은 무엇이 달랐는가?

## 6. 해결 방법

- 원인을 어떻게 확인했는가?
- 어떤 방식으로 고쳤는가?
- 다른 방법 대신 이 방법을 선택한 이유는 무엇인가?

## 7. 검증

실행한 명령어:

```bash
# 예: npm run lint
# 예: npm run build
# 예: npm test
```

확인한 페이지 또는 기능:

- 기능 1
- 기능 2

결과:

- 테스트 통과 여부
- 빌드 통과 여부
- 배포 확인 여부

## 8. 면접에서 말할 포인트

- 이 프로젝트에서 기술적으로 설명할 수 있는 부분
- 문제 해결 과정에서 내가 판단한 부분
- 협업 또는 유지보수 관점에서 좋아진 점
- 수치로 설명할 수 있는 결과

## 9. 다시 한다면 개선할 점

- 구조적으로 더 개선할 수 있는 부분
- 성능, 보안, 테스트, UX 측면에서 보완할 점
- 다음 버전에서 추가하고 싶은 기능

## 10. 짧은 답변 예시

면접에서 30초 안에 말한다면:

> 이 프로젝트는 [문제]를 해결하기 위해 만들었습니다. 처음에는 [어려움]이 있었고, 저는 [내가 한 일]을 통해 [해결 방법]을 적용했습니다. 결과적으로 [검증 결과/개선 효과]를 얻었고, 이 과정에서 [면접 포인트]를 설명할 수 있습니다.
