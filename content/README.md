# Content editing guide

GitHub 웹사이트에서 파일을 추가하거나 수정하는 것만으로 사이트 내용을 업데이트할 수 있습니다.

## 새 노트 추가

1. GitHub에서 `content/notes` 폴더로 이동합니다.
2. `Add file` -> `Create new file`을 누릅니다.
3. 파일 이름을 `my-note-title.md`처럼 적습니다.
4. `content/notes/_README.md`의 양식을 복사해 내용을 채웁니다.
5. `Commit changes`를 누르면 Vercel이 자동으로 배포합니다.

## 새 프로젝트 추가

1. `content/projects` 폴더로 이동합니다.
2. `Add file` -> `Create new file`을 누릅니다.
3. 파일 이름을 `my-project.md`처럼 적습니다.
4. `content/projects/_README.md`의 양식을 복사해 내용을 채웁니다.

## 새 활동 추가

1. `content/activities` 폴더로 이동합니다.
2. `Add file` -> `Create new file`을 누릅니다.
3. 파일 이름을 `my-activity.md`처럼 적습니다.
4. `content/activities/_README.md`의 양식을 복사해 내용을 채웁니다.

## 규칙

- 파일 이름은 영어 소문자, 숫자, 하이픈을 추천합니다.
- 예: `java-study-note.md`, `new-internship-project.md`
- `_README.md`처럼 밑줄로 시작하는 파일은 사이트에 표시되지 않습니다.
- 저장 후 배포까지 보통 1~3분 정도 걸립니다.
